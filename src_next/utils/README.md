# utils/ 通用工具层

> 本层是 `src_next/` 重构架构中的通用工具层。整体架构图、各层关系、数据流请见 [../README.md](../README.md) 的「〇、一图看懂 src_next」。

## 一、这一层负责什么

* 文件读写（txt / 二进制）。
* JSON 保存和加载（带缩进、带校验）。
* 路径管理（输出目录创建、子目录路径拼接）。
* 音频时长读取（用于 RTF 统计）。
* 文本清理（去除多余空白、特殊字符）。
* 安全字符串截断（避免 prompt 超长）。

## 二、这一层不负责什么

* 不包含任何具体业务逻辑。
* 不直接调用模型（不调 LLM、不调 TTS）。
* 不知道当前是哪条 pipeline、哪个 profile。
* 不解析 story / character / director plan 等业务结构。

## 三、输入

* 通用输入：文件路径、JSON 数据、字符串、音频文件等。

## 四、输出

* 通用输出：处理后的数据、保存后的文件路径、读取到的对象等。

## 五、未来会放的文件

```text
utils/
├── io_utils.py     # 文件读写、路径管理
├── json_utils.py   # JSON 保存 / 加载 / 安全序列化
└── audio_utils.py  # 音频时长读取、格式转换
```

## 六、和其他层的交互

* **被所有层共用**：`app/`、`core/`、`analysis/`、`llm/`、`tts/`、`voicebank/` 都可以调用。
* **不主动调用其他业务层**：本层是被动工具集合。
* **不依赖 `profiles/`**：本层是纯工具，不需要知道当前运行环境。
* **保持无状态**：所有函数尽量是纯函数，方便测试和复用。
