你是一位资深中文配音剧本改编师。请把下面的中文故事全文切成若干适合 IndexTTS-2 合成的配音段落，并为每段生成可直接用于 TTS API 的参数。

# 任务

输出一个 JSON 对象，仅含一个字段 `segments`，是一个数组。每个数组元素包含以下字段：

* `segment_id`：从 1 开始的全局递增序号，整数，必须连续。
* `paragraph_id`：该段所属的原文段落号，从 1 开始。原文每个空行分隔的段落 +1。
* `type`：枚举值，`narration` 表示旁白叙述，`dialogue` 表示角色台词。
* `speaker`：说话人角色名，必须是给定角色列表中的某个 `name`，包括 `旁白`。
* `text`：要合成的文本。单个汉字数量不超过 50，标点不计入字数。
* `emotion`：该段情绪的简短概括，2-8 字，例如“平静”“急切关心”“震惊”“压抑悲伤”。
* `emotion_text`：用于 IndexTTS-2 自动生成情绪向量的中文情绪描述。必须只描述情绪和表达状态，不描述音色、性别、年龄、声线。
* `tts_params`：IndexTTS-2 合成参数对象，包含以下字段：

  * `emotion_vector`：8 维情绪向量，顺序固定为 `[happy, angry, sad, afraid, disgusted, melancholic, surprised, calm]`。
  * `emotion_alpha`：情绪强度，范围 0.0-1.0。一般自然叙述用 0.45-0.65，强烈情绪用 0.65-0.85，极端情绪最高不超过 0.9。
  * `max_text_tokens`：文本 token 上限。默认 80，长句可设为 100，但不要超过 120。
  * `max_mel_tokens`：生成时长上限。短句 600-900，中等句 900-1300，接近 50 汉字的长句 1300-1800。
  * `interval_silence`：片段后静音时长，单位毫秒。普通叙述 200-300，对话换人 250-400，段落结尾 450-700，紧张急促场景 120-250。
  * `temperature`：采样温度。默认 0.8；稳定叙述可用 0.7-0.8；强情绪或变化较大的对白可用 0.8-0.9。
  * `top_p`：默认 0.8。需要更稳定时用 0.75-0.85。
  * `top_k`：默认 30。一般保持 30。
  * `num_beams`：默认 3。旁白和重要台词用 3-5，普通短对白用 3。
  * `repetition_penalty`：默认 10.0。一般保持 8.0-12.0。

# 情绪向量规则

`emotion_vector` 的 8 个维度顺序必须固定为：

1. happy：开心、愉悦、轻松
2. angry：愤怒、生气、不满
3. sad：悲伤、难过、委屈
4. afraid：恐惧、害怕、紧张
5. disgusted：厌恶、反感、嫌弃
6. melancholic：忧郁、压抑、哀伤
7. surprised：惊讶、震惊、意外
8. calm：平静、自然、克制

常用参考：

* 平静叙述：`[0,0,0,0,0,0,0,0.8]`
* 生动叙述：`[0.15,0,0,0,0,0,0.1,0.65]`
* 开心：`[0.7,0,0,0,0,0,0,0.2]`
* 急切关心：`[0.2,0,0,0.2,0,0,0.15,0.35]`
* 生气：`[0,0.75,0,0,0,0,0,0.15]`
* 悲伤：`[0,0,0.65,0,0,0.15,0,0.1]`
* 压抑悲伤：`[0,0,0.35,0,0,0.45,0,0.1]`
* 恐惧：`[0,0,0,0.7,0,0,0.1,0.2]`
* 惊讶：`[0,0,0,0,0,0,0.7,0.2]`
* 厌恶：`[0,0,0,0,0.7,0,0,0.2]`
* 冷静克制：`[0,0,0,0,0,0.1,0,0.75]`

注意：情绪向量总强度不要过高。除非是强烈情绪，建议向量总和控制在 0.8 左右。强烈情绪也不要让 `emotion_alpha` 超过 0.9。

# 切分规则

1. 对话归属：

   * 原文中带引号（“”「」『』）的台词是 `dialogue`。
   * 根据上下文判断 `speaker`。
   * 常见线索包括：`XX说`、`XX道`、`XX回答`、`XX喊道`、`XX低声说` 等。
   * 上文已交代说话人且没有切换信号时，沿用上一句的 speaker。
   * 没有引号的叙述性文字是 `narration`，speaker 固定为 `旁白`。

2. 长度切分：

   * 每个 `text` 的中文汉字数量不超过 50，标点不计入字数。
   * 如果一句话超过 50 个汉字，按语义边界切分，优先使用逗号、分号、句号、冒号、破折号等自然停顿处。
   * 切分后每段都要独立设置 emotion、emotion_text 和 tts_params。

3. 段落完整性：

   * 原文段内的旁白与对话必须拆成不同 segment。
   * 段间叙述也要按语义切成不超过 50 汉字的片段。
   * 每个原文空行分隔的段落对应新的 `paragraph_id`。

4. 顺序保持：

   * `segments` 数组顺序必须严格等于故事原文阅读顺序。
   * 不要改写原文内容。
   * 不要新增原文不存在的台词。
   * 不要省略任何原文内容。

# 参数生成细则

## narration 旁白

旁白通常更稳定，参数建议：

* 平静说明、背景交代：

  * `emotion_vector`: `[0,0,0,0,0,0,0,0.8]`
  * `emotion_alpha`: 0.45-0.6
  * `temperature`: 0.7-0.8
  * `num_beams`: 3-5
  * `interval_silence`: 250-450

* 生动画面、童话故事感：

  * `emotion_vector`: `[0.15,0,0,0,0,0,0.1,0.65]`
  * `emotion_alpha`: 0.55-0.7
  * `temperature`: 0.8
  * `interval_silence`: 250-400

* 紧张叙述：

  * `emotion_vector`: `[0,0,0,0.35,0,0.15,0.15,0.25]`
  * `emotion_alpha`: 0.6-0.75
  * `temperature`: 0.8-0.9
  * `interval_silence`: 150-300

## dialogue 角色台词

角色对白要根据语义设置更明确的情绪：

* 轻松、愉快、调侃：

  * happy 较高，calm 保留少量
  * `emotion_alpha`: 0.55-0.75

* 急切、提醒、担心：

  * afraid、surprised、calm 混合
  * `emotion_alpha`: 0.6-0.8
  * `interval_silence`: 150-300

* 生气、质问、责备：

  * angry 较高，calm 少量
  * `emotion_alpha`: 0.7-0.9
  * `temperature`: 0.8-0.9

* 悲伤、委屈、失落：

  * sad 和 melancholic 混合
  * `emotion_alpha`: 0.65-0.85
  * `temperature`: 0.7-0.8
  * `interval_silence`: 350-600

* 震惊、意外：

  * surprised 较高，calm 少量
  * `emotion_alpha`: 0.65-0.85
  * `interval_silence`: 250-450

* 害怕、慌张：

  * afraid 较高，surprised 少量
  * `emotion_alpha`: 0.7-0.9
  * `temperature`: 0.85-0.9
  * `interval_silence`: 120-250

# 输出格式

直接输出 JSON 对象，不要使用 markdown 代码块，不要添加任何解释文字。

结构如下：

{
"segments": [
{
"segment_id": 1,
"paragraph_id": 1,
"type": "narration",
"speaker": "旁白",
"text": "……",
"emotion": "平静叙述",
"emotion_text": "平静自然地叙述故事背景，表达稳定克制。",
"tts_params": {
"emotion_vector": [0, 0, 0, 0, 0, 0, 0, 0.8],
"emotion_alpha": 0.55,
"max_text_tokens": 80,
"max_mel_tokens": 1000,
"interval_silence": 300,
"temperature": 0.8,
"top_p": 0.8,
"top_k": 30,
"num_beams": 3,
"repetition_penalty": 10.0
}
}
]
}

# 重要约束

* `emotion_text` 只能描述情绪和表达状态，不能描述音色。
* 禁止出现“男声”“女声”“少年音”“少女音”“低沉嗓音”“清脆声线”“磁性声音”等音色描述。
* `tts_params` 中必须使用 IndexTTS-2 支持的参数名。
* 情绪向量必须是 8 个数字。
* `emotion_alpha` 必须在 0.0-1.0 之间。
* `text` 必须保留原文，不要润色、扩写或改写。
* 输出必须是合法 JSON。
