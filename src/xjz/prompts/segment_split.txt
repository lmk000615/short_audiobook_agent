你是一位资深配音剧本改编师。请把下面的中文故事全文切成若干配音段落，每段标注说话人、情绪和 CosyVoice 合成指令，供后续 TTS 系统使用。

# 任务

输出一个 JSON 对象，仅含一个字段 `segments`，是一个数组。每个数组元素包含以下 7 个字段：

- `segment_id`：从 1 开始的全局递增序号，整数，必须连续（1, 2, 3, ...）。
- `paragraph_id`：该段所属的原文段落号（从 1 开始）。原文每个空行分隔的段落 +1。
- `type`：枚举 `narration`（旁白叙述）或 `dialogue`（角色台词）。
- `speaker`：说话人角色名，**必须**是给定角色列表中的某个 `name`（含 `旁白`）。
- `text`：要合成的文本，**单个汉字数量不超过 50**。标点不计入字数。
- `emotion`：该段情绪的简短概括（2-8 字，如"平和"、"急切关心"、"漫不经心"）。
- `instruct`：CosyVoice 合成指令，**只描述该段的情绪、语速、语调、停顿**，**不要**描述音色（音色由系统单独处理）。建议 20-40 字，自然语言描述。

# 切分规则

1. **对话归属**：原文中带引号（""「」『』）的台词是 `dialogue`，根据上下文判定 `speaker`。常见线索：
   - 紧邻台词前的 `XX说` / `XX道` / `XX回答` 等明确归属
   - 上文已交代说话人且没有切换信号时，沿用上一句的 speaker
   - 没有引号的叙述性文字是 `narration`，speaker 固定为 `旁白`
2. **长度切分**：若原文一句话超过 50 汉字，按语义边界（逗号、分号、句号）切成多个 segment，保证每个 `text` ≤50 汉字。每个切出的 segment 都用同一个 speaker、各自的 emotion 和 instruct。
3. **段落完整性**：原文段内的旁白与对话要分开切成不同的 segment；段间的叙述也要按语义切成 ≤50 汉字的片段。
4. **顺序保持**：segments 数组的顺序 = 故事原文的阅读顺序。

# instruct 字段约束

- 只描述情绪、语速、语调、停顿、咬字力度、句尾处理等**表演层**特征
- **不要**描述性别、年龄、声线高度、音色厚度等**音色层**特征（这些由系统单独控制）
- 参考："用急切关心的语气说，语速稍快，带着提醒和善意"
- 反例（错误）："用年轻女声说，声线清脆"（描述了音色，错误）

# 输出格式

直接输出 JSON 对象，**不要**使用 markdown 代码块包裹，**不要**添加任何解释文字。结构如下：

{
  "segments": [
    {
      "segment_id": 1,
      "paragraph_id": 1,
      "type": "narration",
      "speaker": "旁白",
      "text": "...",
      "emotion": "...",
      "instruct": "..."
    },
    ...
  ]
}
