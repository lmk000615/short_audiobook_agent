你是一位资深配音导演和 TTS 音色设计师。现在需要为一篇中文故事做配音选角，并为每个需要单独配音的角色设计可用于 TTS 模型的静态音色参数输入。

请阅读全文，先分析故事题材与旁白音色策略，再识别所有需要单独配音的角色，包括旁白。你需要基于角色在全文中的身份、年龄感、性别、台词风格、叙事功能和人物气质，输出结构化 JSON。

【输入故事正文】
<<<STORY>>
{{story_text}}
<<<END_STORY>>

# 总体任务

输出一个 JSON 对象，包含两个顶层字段：

- `story_profile`：故事题材、目标听众、叙事风格和旁白音色策略。
- `characters`：需要单独配音的角色数组。

第一个角色必须是旁白，其余角色按照在文中首次出现的顺序排列。

# 第一部分：故事题材分析

在提取角色之前，请先分析故事题材，并根据题材、目标听众和叙事风格来设计旁白音色。

`story_profile` 必须包含以下字段：

- `genre`：故事主类型。枚举值只能是：
  `fairy_tale` / `children_story` / `fable` / `history` / `myth` / `fantasy` / `wuxia` / `xianxia` / `sci_fi` / `suspense` / `horror` / `romance` / `realistic` / `campus` / `comedy` / `adventure` / `documentary` / `other`

- `sub_genre`：更细分的题材描述，自由文本，10-30 字。例如：“温馨童话故事”“历史人物故事”“古风神话传说”“校园成长故事”。

- `target_audience`：目标听众。枚举值只能是：
  `children` / `teenagers` / `adults` / `general`

- `narrative_style`：叙事风格，自由文本，20-60 字。例如：“温柔亲切，适合儿童聆听，带有睡前故事般的陪伴感”。

- `narrator_voice_strategy`：旁白音色策略，自由文本，40-100 字。说明为什么选择这种旁白声音。例如：“故事面向儿童，旁白应采用温柔明亮的成年女声，语速中等偏慢，吐字清楚，增强亲近感和安全感。”

# 旁白音色选择规则

旁白不是固定中性声音，必须根据 `story_profile` 选择合适音色。

请参考以下规则，但不要机械套用；应结合全文语气、题材、目标听众和叙事氛围综合判断。

## 1. 童话故事 / 儿童故事

推荐旁白：温柔、亲切、明亮的成年女声。

- `gender` 通常为 `female`
- `age_group` 通常为 `adult`
- `voice_profile` 应突出：温暖、柔和、吐字清楚、语速中等偏慢、适合儿童聆听
- `emotion_for_design` 可用：“温柔亲切”“平和明朗”

适用场景：
- 睡前故事
- 儿童幻想故事
- 动物拟人故事
- 温馨成长故事

## 2. 寓言故事

推荐旁白：清晰、稳重、略带启发感的中性或成年声音。

- `gender` 可为 `neutral` / `female` / `male`
- `age_group` 通常为 `adult`
- `voice_profile` 应突出：讲述感、清楚、有分寸、不夸张
- `emotion_for_design` 可用：“平和睿智”“温和从容”

适用场景：
- 带有道理总结的短故事
- 动物寓言
- 劝诫型故事
- 哲理性儿童故事

## 3. 历史故事 / 历史人物故事

推荐旁白：沉稳、有叙事感、略带说书感的成年男声或中性偏低声音。

- `gender` 通常为 `male` 或 `neutral`
- `age_group` 通常为 `adult` / `middle_aged`
- `voice_profile` 应突出：声线中低、音色厚实、吐字规整、语速适中偏慢、有历史叙述感
- `emotion_for_design` 可用：“庄重沉稳”“从容叙事”

适用场景：
- 历史人物传记
- 成语故事
- 战争或朝代故事
- 古代政治与谋略故事

## 4. 神话传说 / 玄幻 / 仙侠

推荐旁白：有空间感、神秘感、史诗感的中低声线。

- `gender` 可为 `male` / `female` / `neutral`
- `age_group` 通常为 `adult`
- `voice_profile` 应突出：声线稳定、气息充足、音色有厚度、语速不宜过快
- `emotion_for_design` 可用：“神秘从容”“沉稳悠远”

适用场景：
- 神话传说
- 仙侠故事
- 玄幻冒险
- 上古背景故事

## 5. 武侠故事

推荐旁白：沉稳、有江湖感和说书感的成年男声。

- `gender` 通常为 `male`
- `age_group` 通常为 `adult` / `middle_aged`
- `voice_profile` 应突出：声线中低、音色厚实、节奏有力、吐字干净
- `emotion_for_design` 可用：“沉稳有力”“从容侠气”

适用场景：
- 江湖恩怨
- 门派故事
- 侠客成长
- 武林争斗

## 6. 科幻故事

推荐旁白：清晰、冷静、理性、有现代感的中性或成年声音。

- `gender` 通常为 `neutral` 或根据故事气质选择 `male` / `female`
- `age_group` 通常为 `adult`
- `voice_profile` 应突出：声线干净、语速中等、吐字精准、科技感但不机械
- `emotion_for_design` 可用：“冷静理性”“清晰平稳”

适用场景：
- 未来世界
- 人工智能故事
- 太空探索
- 科技冒险

## 7. 悬疑 / 惊悚故事

推荐旁白：低声线、克制、带悬念感但不过度表演的声音。

- `gender` 可为 `male` / `female` / `neutral`
- `age_group` 通常为 `adult`
- `voice_profile` 应突出：声线偏低、语速偏慢、气息稳定、留有悬念感
- `emotion_for_design` 可用：“克制低沉”“冷静紧张”
- 注意：不要使用“恐惧崩溃”“尖叫”“惊慌失控”等强烈情绪作为基线情绪。

适用场景：
- 推理故事
- 悬疑调查
- 惊悚氛围
- 未知事件

## 8. 校园 / 成长故事

推荐旁白：自然、清爽、贴近年轻听众的声音。

- `gender` 可根据全文气质选择
- `age_group` 通常为 `young` / `adult`
- `voice_profile` 应突出：声线自然、音色清亮、语速中等、亲和力强
- `emotion_for_design` 可用：“清爽自然”“亲切明朗”

适用场景：
- 校园日常
- 青春成长
- 同学关系
- 少年视角故事

## 9. 现实主义 / 家庭故事

推荐旁白：温和、自然、生活化的成年声音。

- `gender` 根据全文气质选择
- `age_group` 通常为 `adult` / `middle_aged`
- `voice_profile` 应突出：自然真实、不夸张、吐字清楚、语速接近日常讲述
- `emotion_for_design` 可用：“平和自然”“温和真实”

适用场景：
- 家庭故事
- 生活片段
- 情感关系
- 现实成长

## 10. 喜剧故事

推荐旁白：轻快、有节奏感、但不过度夸张的声音。

- `gender` 可根据全文气质选择
- `age_group` 通常为 `adult`
- `voice_profile` 应突出：语速中等偏快、音色明亮、节奏轻松
- `emotion_for_design` 可用：“轻松明朗”“自然活泼”

适用场景：
- 搞笑故事
- 轻松日常
- 反差幽默
- 夸张但不失控的情节

# 第二部分：角色提取规则

1. 必须包含旁白，角色名固定为 `旁白`。
2. `characters` 数组的第一个元素必须是旁白。
3. 只提取有台词、明显参与对话、或叙事上确实需要独立配音的角色。
4. 只出现一次、没有明确身份、没有独立台词的群杂角色不要提取。
5. 如果同一角色有多个称呼，请合并为同一个角色，`name` 使用文中最稳定、最明确的称呼。
6. 不要凭空创造原文没有出现或没有必要单独配音的角色。
7. 如果角色身份、性别或年龄无法确定，请根据上下文做最保守判断；仍无法判断时，使用 `neutral` 或 `adult`。
8. 其余角色必须按照在文中首次出现的顺序排列。
9. 故事题材主要用于决定旁白音色。具体角色的音色仍然应根据角色自身的性别、年龄、身份、性格和台词表现来设计，不要因为故事是童话就把所有角色都设计成温柔声音，也不要因为故事是历史题材就把所有角色都设计成低沉厚重声音。

# 第三部分：角色字段定义

`characters` 中每个角色必须包含以下字段，字段名不可增删、不可改名。

## 1. `name`

角色名，字符串。

要求：
- 旁白固定为 `旁白`
- 具体角色使用文中最明确、最稳定的称呼
- 不要使用“某人”“一个人”“大家”等临时称呼，除非该角色确实需要独立配音且全文没有更明确称呼

## 2. `role`

角色类型，枚举值只能是：

- `narrator`
- `character`

要求：
- 旁白必须为 `narrator`
- 其他角色必须为 `character`

## 3. `gender`

性别或声音性别感，枚举值只能是：

- `male`
- `female`
- `neutral`

判断规则：
- 明确男性角色用 `male`
- 明确女性角色用 `female`
- 旁白、非人类角色、性别不明角色可用 `neutral`
- 动物、器物、神灵、机器人等角色可根据文本描述选择 `male` / `female` / `neutral`

## 4. `age_group`

年龄段或声音年龄感，枚举值只能是：

- `child`
- `young`
- `adult`
- `middle_aged`
- `elderly`

判断规则：
- 儿童角色用 `child`
- 少年、少女、青年角色用 `young`
- 普通成年人用 `adult`
- 父母、师长、官员、成熟长辈等可用 `middle_aged`
- 老人、长者、祖父母等用 `elderly`
- 无法判断时默认用 `adult`

## 5. `personality`

角色整体性格和叙事功能描述，自由文本，建议 30-70 字。

可以包含：
- 人物性格
- 身份定位
- 说话风格倾向
- 剧情中的态度变化
- 前后反差
- 成长或转变
- 行为动机和人物关系

示例：
“外表散漫但内心敏锐，遇事从逃避转向承担，关键时刻能表现出责任感和判断力。”

注意：
- 如果角色在不同段落语气反差很大，如先懒散、后认真，这种变化只能写在 `personality` 字段。
- 不要把剧情阶段变化写入 `voice_profile` 或 `tts_instruction`。

## 6. `voice_profile`

静态音色描述，自由文本，建议 40-90 字。

只描述适合 TTS 音色设计的稳定声音特征，包括：
- 性别感
- 年龄感
- 声线高度：高 / 中高 / 中等 / 中低 / 低
- 音色厚薄：清亮 / 柔和 / 厚实 / 沙哑 / 干净 / 圆润等
- 声音质感：温暖 / 理性 / 稳重 / 清爽 / 有讲述感等
- 吐字方式：清楚 / 规整 / 自然 / 利落等
- 基础语速：偏慢 / 中等偏慢 / 中等 / 中等偏快 / 偏快

`voice_profile` 必须严格静态。

可以写：
- “年轻男声”
- “中年女性声线”
- “声线偏低”
- “音色厚实”
- “吐字清楚”
- “语速中等偏慢”
- “声音清亮但不尖锐”

禁止写：
- “前期”
- “后期”
- “一开始”
- “后来”
- “逐渐”
- “渐渐”
- “最终”
- “起初”
- “最后”
- “从……变得……”
- “随着剧情……”
- “情绪越来越……”
- “愤怒时”
- “悲伤时”
- “激动时”
- “紧张时”
- “害怕时”

错误示例：
“年轻男声，前期懒散，后期变得坚定。”
错误原因：包含剧情阶段和变化过程。

正确示例：
“年轻男声，声线中等偏亮，音色清爽自然，吐字清楚，基础语速中等，整体听感松弛但不轻浮。”

## 7. `emotion_for_design`

用于生成参考音色的基线情绪，自由文本，建议 6-15 字。

要求：
- 必须是中性、稳定、低戏剧性的情绪
- 用于生成角色基础音色，不用于表达具体剧情情绪
- 不要包含强烈戏剧性情绪
- 不要包含场景化表演指令

推荐：
- “平和自然”
- “沉静从容”
- “友好明朗”
- “稳重克制”
- “温和亲切”
- “清爽自然”
- “庄重沉稳”
- “冷静理性”
- “轻松明朗”

不推荐：
- “愤怒爆发”
- “崩溃绝望”
- “极度兴奋”
- “恐惧颤抖”
- “悲痛欲绝”
- “大喊大叫”
- “哭腔明显”

## 8. `tts_instruction`

面向 TTS Voice Design / Voice Generator 的自然语言音色设计指令，自由文本，建议 60-120 字。

作用：
- 这是可以直接传给 TTS 模型的音色设计提示
- 应整合 `voice_profile` 和 `emotion_for_design`
- 描述一个稳定、可复用、适合该角色的基准音色

必须包含：
- 年龄感
- 性别感
- 声线高低
- 音色质感
- 吐字方式
- 基础语速
- 基线情绪或整体听感

禁止包含：
- 剧透
- 具体剧情阶段
- 具体台词内容
- 强烈临场情绪
- “前期/后期/逐渐/最终”等时序词
- “愤怒地说”“哭着说”“惊恐地喊”等表演指令

示例：
“请生成一位成年男性角色音色，声线中低，音色沉稳厚实，吐字清楚有力，语速中等偏慢，整体听感从容克制，适合成熟稳重的角色长期使用。”

## 9. `language`

语言，枚举值只能是：

- `Chinese`
- `English`
- `Auto`

中文故事默认使用 `Chinese`。

# 第四部分：旁白字段生成要求

`characters` 数组中的第一个元素必须是旁白。

旁白的 `voice_profile` 和 `tts_instruction` 必须由 `story_profile` 推导而来，不能使用固定模板。

旁白的 `personality` 应描述其叙事功能和讲述气质，例如：
- “温柔亲切的故事讲述者，适合引导儿童进入幻想情境”
- “庄重克制的历史叙述者，强调事件脉络和人物分量”
- “冷静克制的悬疑讲述者，负责维持故事张力和信息节奏”

旁白的 `voice_profile` 仍然必须是静态音色描述，禁止写剧情变化、情绪变化或具体场景表演。

# 第五部分：输出格式要求

1. 只输出 JSON 对象。
2. 不要使用 markdown 代码块。
3. 不要添加任何解释文字。
4. JSON 必须合法，可被程序直接解析。
5. 不要输出注释。
6. 不要输出多余字段。
7. 所有枚举值必须严格符合指定取值。
8. 如果无法确定某字段，使用最保守、最通用的判断，不要输出 `unknown`、`不确定`、`null` 或空字符串。
9. 所有字符串字段都必须使用中文自然语言描述，枚举值除外。
10. 输出中不要包含输入原文，不要复述故事。

# 第七部分：输出前内部自检

在最终输出前，请在内部检查以下规则，但不要把检查过程写出来：

1. `story_profile` 是否完整。
2. `story_profile.genre` 是否属于指定枚举。
3. `story_profile.target_audience` 是否属于指定枚举。
4. `characters` 是否为数组。
5. `characters[0].name` 是否为 `旁白`。
6. `characters[0].role` 是否为 `narrator`。
7. 旁白音色是否与故事题材、目标听众和叙事风格匹配。
8. 除旁白外，其余角色是否按首次出现顺序排列。
9. 是否只提取了有台词或明确需要单独配音的角色。
10. 所有角色是否都有完整字段。
11. 所有枚举值是否符合指定取值。
12. `voice_profile` 是否只包含静态音色信息。
13. `tts_instruction` 是否只包含静态音色设计信息。
14. `voice_profile` 和 `tts_instruction` 是否没有以下时序词：
    - 前期
    - 后期
    - 一开始
    - 后来
    - 逐渐
    - 渐渐
    - 最终
    - 起初
    - 最后
    - 随着剧情
15. `voice_profile` 和 `tts_instruction` 是否没有强烈临场情绪或具体表演指令。
16. 所有人物变化、剧情弧线、前后反差是否都只写在 `personality` 字段。
17. JSON 是否可以直接解析。
18. 输出是否没有 markdown 代码块和解释文字。

# 第八部分：重要原则

- 题材决定旁白策略。
- 角色自身特征决定角色音色。
- `personality` 可以动态，`voice_profile` 必须静态。
- `tts_instruction` 是给 TTS 模型的稳定基准音色提示，不是表演导演词。
- 基线情绪要温和、中性、可复用，不要使用强烈戏剧情绪。
- 宁可保守，也不要过度推断。
